SQL -STRUCTURED QUERY LANGUAGE

In the world of Database Management System SQL plays a vital role in the retrieval and management of data. SQL stands for Structured Query Language. As clear from the above-extended form SQL this language helps in database management through user input queries. This tutorial is basically to make the viewers understand and get accustomed to SQL. So lets get started.

INTRODUCTION AND APPLICATIONS

SQL is very proficient in the field of Relational Database Management System like Oracle, MS SQL, MS Access, SQL Servers, etc. Uses SQL as their query language. The applications of SQL are vast. Some of them include:-

  • Allows user to retrieve the data.
  • Allows user to manipulate the data.
  • Allows user to connect with the database.
  • Allows the user to delete or drop databases.

INITIAL STEPS TO START WORKING WITH SQL

First and foremost step is creation of database.The steps for creating database is shown below:

  • CREATE DATABASE DATABASE_NAME;
  • After creation we need to enter into it.
  • USE DATABASE_NAME;
  • This will help us to create a database and then use it.We can create a table after entering into the database.

BASIC COMMANDS USED IN SQL

A classical SQL architecture is shown below in the diagram. CLICK TO KNOW MORE.... We can interact with the database with the help of SQL, and SQL does it by implementing some commands. Some SQL commands are as follows:-

  • CREATE- Creates a new table or other projects in Databases
  • SELECT- Selects certain records from one or more tables.
  • INSERT- Creates a record.
  • UPDATE- It modifies a record
  • DELETE- Deletes record.
  • DROP- Deletes the entire table or a view of a table.

So these were some of the basic commands of SQL needed while manipulating with the database.

DESCRIPTION OF BASIC STRUCTURE

Let us now talk about RDBMS. RDBMS stands for Relational Database Management System. RDBMS acts as the base for SQL (not only SQL but also other database systems). Now let us discuss some of the terms in the fields of RDBMS.
